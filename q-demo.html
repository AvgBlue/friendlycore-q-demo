How to study with this page

This demo is a small playground for the FriendlyCore filtering step. Pick one point, choose how many friends it has, and watch how the function q(c) decides whether that point is kept in the core C.

1. Set n

n is the dataset size. Increasing n moves the cutoff n/2, so a point needs more friends before it has any chance to be kept.

2. Set c

c is the friend count for the point you are testing. Green dots are friends; red dots are not friends.

3. Run a sample

The button draws one random number. If that number is below q(c), the point enters C; otherwise it is removed.

Linear The easiest shape: every extra friend above n/2 increases q by the same amount.
Smoothstep A gentle S-curve. It starts slowly, rises in the middle, and slows down near q=1.
Slow start Stricter near the cutoff. A point needs to be strongly supported before q becomes large.
Fast start More generous after the cutoff. Once c is above n/2, q rises quickly.

The important lecture idea is not the exact curve; it is the smooth transition. A hard jump from 0 to 1 could reveal too much when one record is added or removed. A smooth q(c) makes the filtering step stable, which is why FriendlyCore can safely remove outliers before private averaging or clustering.
